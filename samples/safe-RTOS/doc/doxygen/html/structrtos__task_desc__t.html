Detailed Description

Specification of a task function. Objects of this type are used internally by the scheduler for the regularly or event triggered user mode and OS tasks but also visibly by the client code, when running a user mode task using rtos_runTask().

Field Documentation

◆ addrTaskFct

uintptr_t rtos_taskDesc_t::addrTaskFct

Address of task function, which is run in user mode and with process ID PID.
Note, the signature of a task function differs. An initialization task and a user mode task function can signal an error by return value, while a scheduled OS task function can't. A task function started with rtos_runTask() or with rtos_osRunTask() has a caller provided argument, which is not available to regularly scheduled tasks.
In the assembler code, this field is addressed to by offset O_TCONF_pFct.

◆ tiTaskMax

uint32_t rtos_taskDesc_t::tiTaskMax

Time budget for the user task in ticks of the time base (i.e., TBL or STM depending on MCU derivative). This budget is granted for each activation of the task. The budget relates to deadline monitoring, i.e., it is a world time budget, not an execution time budget.
Macros RTOS_TI_MS2TICKS and RTOS_TI_US2TICKS can be used to state the time budget in Milli- or Mircoseconds.
A value of zero means that deadline monitoring is disabled for the task.
In the assembler code, this field is addressed to by offset O_TCONF_tiMax.

◆ PID

uint8_t rtos_taskDesc_t::PID

The process ID of the userTask in the range 1..RTOS_NO_PROCESSES (PID 0 is reserved for kernel operation). At the same time index into the array of process descriptors.
In the assembler code, this field is addressed to by offset O_TCONF_pid.
