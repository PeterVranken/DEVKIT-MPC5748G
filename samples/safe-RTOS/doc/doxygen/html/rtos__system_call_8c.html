Detailed Description

Implementation of the system call table as a const data object in ROM. The information about the table contents is taken from dedicated header files, named *_defSysCalls.h, which have to be provided by all units that offer at least one system call.

#define RTOS_SYSCALL_DUMMY_TABLE_ENTRY    RTOS_SC_TABLE_ENTRY(rtos_scBscHdlr_sysCallUndefined, BASIC)

This table entry is used for those system table entries, which are not defined by any included I/O driver header file. The dummy table entry points to a no-operation service, which silently returns to the caller.

◆ SYS_CALL_TABLE_DATA

#define SYS_CALL_TABLE_DATA (   idxCore)

Initializer expression for the data object that implements the system call table of a single core.
Note, the initializer expression requires maintenance if the size RTOS_NO_SYSTEM_CALLS of the system call table is changed
